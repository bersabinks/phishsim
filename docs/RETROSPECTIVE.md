# Rétrospective de Sprint

Trame à compléter en équipe. L'objectif est de produire **une amélioration
concrète et mesurable**, pas un simple tour de table.

## 1. Format : « Continuer / Arrêter / Commencer »

### Continuer
- _(ex.)_ Livrer un parcours vertical démontrable avant tout durcissement.
- _(ex.)_ Écrire les tests en même temps que le code.

### Arrêter
- _(ex.)_ Empiler des fonctionnalités d'extension avant l'acceptation du MVP.

### Commencer
- _(ex.)_ Tracer systématiquement chaque contribution de l'IA dans le message de
  commit (objectif + correction humaine).

## 2. Ce qui a bien fonctionné

- Le découpage en stories ordonnées a permis une démonstration rapide.
- Le garde-fou « pas d'e-mail réel » a clarifié la frontière éthique dès le début.

## 3. Ce qui a été difficile

- L'arbitrage de périmètre sous contrainte de 15 h.
- _(à compléter par l'équipe)_

## 4. Amélioration retenue (une seule, mesurable)

| Élément | Détail |
|---|---|
| **Action** | _(ex.)_ Ajouter une étape de couverture de tests dans la CI |
| **Indicateur** | _(ex.)_ Couverture de code mesurée et affichée à chaque exécution |
| **Cible** | _(ex.)_ ≥ 80 % de couverture sur `app/services.py` |
| **Responsable** | _(à désigner)_ |
| **Échéance** | _(prochaine itération)_ |

## 5. Vérification au prochain sprint

- L'indicateur retenu est-il atteint ?
- L'action a-t-elle été réellement mise en œuvre ?
